# FundPilot_SYM 프로젝트 동작 개요

이 문서는 현재 FundPilot_SYM 데모가 `기업입력 -> 분석 -> TOP3 비교 -> 공고신청` 흐름에서 어떤 데이터를 받고, 어떤 방식으로 추천과 승인 가능성을 계산하며, 사용자를 실제 공고로 연결하는지 정리한 문서입니다.

## 1. 프로젝트 목적

FundPilot_SYM은 FundPilot의 프로토타입입니다. 중소벤처기업진흥공단 정책자금 공공데이터와 정책자금별 주요 신청 조건을 활용해 기업 특성에 맞는 정책자금을 추천하고, 승인 가능성 및 제한/주의 사유를 보여주는 Streamlit 기반 데모 앱입니다.

주요 소스 파일은 다음과 같습니다.

| 파일 | 역할 |
| --- | --- |
| `app/app.py` | Streamlit UI, 입력폼, 추천 룰, 화면 전환, 공고 링크 생성 |
| `app/approval_model.py` | 공공데이터 기반 합성 학습데이터 생성 및 RandomForest 승인 가능성 예측 |
| `app/data_store.py` | CSV 원천자료를 SQLite DB로 적재하고 앱/모델에 데이터셋 제공 |
| `app/api_clients.py` | 공공데이터/중소벤처24 API 인증 환경 로딩, ODcloud 및 일반 공공데이터포털 GET 호출, 수집 로그/원문 캐시 연계 |
| `data/*.csv` | 업종별, 매출액별, 업력별 등 중진공 정책자금 지원 실적 데이터 |
| `data/*.pdf` | 기준금리 등 정책자금 분석에 참고하는 원천자료 |
| `data/fundpilot.db` | 실행 시 자동 생성되는 SQLite 로컬 캐시 DB |
| `docs/Kosmes_input_fields.md` | 실제 중진공 정책자금 신청 입력 항목 정의서 |
| `docs/api.md` | 외부 API별 AI 가이드 문서 경로 인덱스 |
| `docs/api_integration_summary.md` | 개발 에이전트가 참고할 API 연동 우선순위, 저장 후보, 구현 체크리스트 |
| `docs/api_utilization_matrix.md` | `docs/api.md`의 모든 API를 활용 목적, 적용 영역, 저장 후보, 구현 상태로 추적하는 필수 적용 매트릭스 |
| `docs/api_guides/*.md` | 중소벤처24, 금융위원회, 중진공/ODcloud API 연동을 위한 AI용 상세 가이드 |

데이터 파일은 프로젝트 루트의 `data/` 폴더에 모아 관리합니다. `app/data_store.py`는 실행 위치와 무관하게 `Path(__file__)` 기준으로 프로젝트 루트를 찾고, 그 아래 `data/`의 CSV 파일을 `data/fundpilot.db`에 적재합니다. `app/app.py`와 `app/approval_model.py`는 CSV를 직접 읽지 않고 `data_store.read_dataset()`을 통해 SQLite DB에서 데이터를 가져옵니다.

현재 SQLite에는 다음 성격의 테이블이 생성됩니다.

| 테이블 | 원천자료 |
| --- | --- |

| `kosmes_policy_fund_excluded_industries` | 중진공 융자제외 대상 업종 API 응답을 기준일, 업종분류, 업종코드, 업종명으로 정규화한 테이블 |

### 1-1. 외부 API 가이드 문서

향후 하드코딩된 데이터와 수동 CSV를 Open API 기반 수집으로 대체하기 위해 `docs/api.md`에 API 이름과 가이드 문서 경로를 인덱싱합니다. `docs/api.md`에 등록된 모든 API는 최종 시스템 적용 대상이며, `docs/public_data_api_candidates.md`에 적힌 FundPilot 활용 목적에 맞게 기업입력, 분석, TOP3 비교, 공고신청, 리스크 중 하나 이상의 실제 기능에 연결해야 합니다. 적용 상태는 `docs/api_utilization_matrix.md`에서 API별로 추적합니다. 현재 `docs/api_guides/`에는 중소벤처24 공고/증명서, 금융위원회 기업 기본·재무 정보, 중진공 정책자금 지원 현황 계열 API 가이드가 정리되어 있습니다.

Swagger URL만 제공되던 중진공/ODcloud 데이터는 API별 Markdown 가이드로 변환해 저장했습니다. 이 가이드들은 실제 API 키를 포함하지 않고, 인증키는 `DATA_GO_KR_SERVICE_KEY` 같은 환경변수 이름 또는 `{DATA_GO_KR_SERVICE_KEY}` 자리표시자로만 설명합니다.

현재 가이드화된 중진공/ODcloud 계열 데이터는 종업원 규모별, 자산 규모별, 업종별, 업력별, 자금종류별, 권역/지역별, 제조현장스마트화, 이차보전, 청년전용창업, 창업기반지원, 수출기업화, 재창업, 비대면창업, 융자제외 대상 업종 등입니다. 이 데이터들은 향후 추천 점수 보정, 표준 수혜 패턴 산출, 리스크 비교, 융자제외 업종 판정에 활용할 후보입니다.

`docs/api_integration_summary.md`는 위 API들을 개발 순서, FundPilot 단계별 활용, DB 저장 후보, 공통 구현 체크리스트 기준으로 다시 묶은 통합 참고 문서입니다.

API 연동 공통 기반은 `app/api_clients.py`와 `app/data_store.py`에 분리되어 있습니다. `.env`는 `app/.env`와 프로젝트 루트 `.env`를 모두 탐색하며, 현재 사용하는 환경변수 이름은 `SMES24_OPENAPI_TOKEN`, `DATA_GO_KR_SERVICE_KEY`입니다. ODcloud 계열 API는 `odcloud_get()`이 `page`, `perPage`, `returnType`, `serviceKey` 기본 파라미터로 호출하고, 일반 공공데이터포털 API는 `public_data_get()`이 `pageNo`, `numOfRows`, `resultType=json`, `serviceKey` 기본 파라미터로 호출합니다. 두 helper 모두 인증키를 로그나 캐시 키에 남기지 않고, 수집 결과는 `success`, `missing_key`, `http_error`, `failed`, `empty` 상태로 `api_fetch_logs`에 남깁니다. 첫 등록 대상은 최신 2026-01-01 기준 `중소벤처기업진흥공단_정책자금 융자제외 대상 업종` API입니다. 해당 API는 성공적으로 수집되면 원문 캐시와 별도로 `kosmes_policy_fund_excluded_industries` 정규화 테이블에 저장되고, 앱의 융자제외 업종 판정은 이 API 테이블을 우선 사용한 뒤 기존 CSV 테이블로 대체합니다.

P1 기업정보 보강은 STEP 1 입력 화면에도 연결되어 있습니다. 금융위원회 `getCorpOutline_V2`는 사용자가 기업명 또는 법인등록번호로 `기업정보 조회` 버튼을 누를 때만 `get_corp_outline_v2(corp_name 또는 crno)`로 호출되며, 응답 후보를 사용자가 선택한 뒤 `선택 기업정보 자동입력` 버튼을 눌러야 입력폼에 반영됩니다. 기업명, 사업자등록번호, 창업일, 상시근로자 수, 증권시장 상장 여부를 보수적으로 채우고, 업종명/주요사업/등록시장 정보는 기존 업종 드롭다운을 직접 덮어쓰지 않고 안내 문구로 표시합니다. 선택 후보에 법인등록번호가 있으면 최근 3개년 기준으로 `get_summ_fina_stat_v2(crno, biz_year)`를 호출해 매출, 영업이익, 자산총계, 자본총계, 부채비율 자동입력을 시도합니다. API 응답이 없거나 실패하면 안내 메시지만 표시하고 기존 수동 입력값을 유지합니다.

## 2. 전체 사용자 흐름

앱은 Streamlit `session_state.step` 값으로 4단계 화면을 제어합니다.

| 단계 | 화면 | 핵심 기능 |
| --- | --- | --- |
| STEP 1 | 기업 정보 입력 | 기업 기본정보, 재무, 신용, 수혜이력, 정책우선도, 희망 자금 입력 |
| STEP 2 | 분석 결과 | AI 승인 가능성, 최우선 추천 자금, 조건/한도/금리 표시 |
| STEP 3 | TOP3 비교 | 추천 정책자금 3개를 점수와 조건 중심으로 비교 |
| STEP 4 | 공고 신청 | 추천 자금별 중소벤처24 공고 검색 링크 제공 |

## 3. STEP 1: 기업 정보 입력

현재 기업 입력 화면은 `docs/Kosmes_input_fields.md`의 실제 신청 항목을 반영해 기존 단순 입력폼보다 확장되어 있으며, 화면 상단에는 기업정보 자동조회 영역이 있습니다. 자동조회는 앱 import 시 실행되지 않고 사용자가 버튼을 누를 때만 API를 호출합니다.

### 3-1. 기업 기본 정보

입력받는 항목은 다음과 같습니다.

| 항목 | 입력 방식 | 사용 목적 |

| 파생 업력 | 자동 계산 | 업력별 공공데이터 컬럼 및 자금 조건 판별 |

### 3-3. 신용·세금 및 수혜 이력

이 섹션은 신청 제한에 직접 영향을 주는 항목이 많습니다.

| 항목 | 사용 방식 |
| --- | --- |

## 4. STEP 2: 분석 및 승인 가능성 예측

분석 단계는 크게 두 갈래로 동작합니다.

1. 정책자금 추천 룰 엔진
2. RandomForest 기반 승인 가능성 예측

### 4-1. RandomForest 승인 가능성 예측

승인 가능성 예측은 `app/approval_model.py`의 `predict_approval()` 함수를 통해 수행됩니다. 이 함수는 내부적으로 `predict_fit_score()`의 별칭입니다.

모델 학습 방식은 다음과 같습니다.

| 단계 | 설명 |
| --- | --- |
| 공공데이터 로드 | SQLite DB에 적재된 업종별, 업력별, 매출액별 지원 실적 테이블을 읽음 |
| 가중치 생성 | 각 업종/업력/매출 구간의 지원금액 비율을 계산 |
| 합성 학습데이터 생성 | 공공데이터 분포를 반영해 3,000건의 synthetic 데이터를 생성 |
| 규칙 기반 라벨 생성 | 기술 보유, 제조업, 청년, 결격요인 등을 반영해 적합/부적합 라벨 생성 |
| 모델 학습 | `RandomForestClassifier`로 적합도 분류 모델 학습 |
| 확률 예측 | `predict_proba()`의 적합 클래스 확률을 0~100%로 반환 |

모델 입력값은 다음과 같습니다.

| 입력값 | 설명 |
| --- | --- |
| 업종 | 금속, 기계, 전기, 전자 등 내부 인코딩 |
| 매출 규모 | 5억 미만부터 300억 이상까지 구간 인코딩 |
| 업력 | 창업일에서 계산한 업력 구간 |
| 자금 목적 | 운전자금, 시설자금, 창업자금, 기술개발 등 |
| 기업 형태 | 예비창업자, 개인사업자, 법인사업자, 소상공인, 중소기업 |
| 수출 단계 | 내수기업, 수출 준비, 수출 10만 달러 미만/이상 |
| 기술 보유 여부 | 특허, R&D, 인증, IP 등으로 판정 |
| 제조업 여부 | KSIC 또는 체크값으로 판정 |
| 대표자 연령 | 청년창업 조건 반영 |
| 체납 여부 | 결격 요인 |
| 신용제한 여부 | 결격 요인 |

### 4-2. 승인 가능성 보정

RandomForest 결과는 `app/app.py`의 `adjust_approval_probability()`에서 한 번 더 보정됩니다.

| 조건 | 보정 방식 |
| --- | --- |
| 하드 블록이 1개 이상 | 승인 가능성 상한 20% |
| 재무 제한 위험 | 승인 가능성 상한 45% |
| 소상공인 제한 가능성 | 승인 가능성 상한 60% |
| 비수도권 | 가산 |
| 신규 고용 10인 이상 | 가산 |
| 3개년 매출 성장률 20% 이상 | 가산 |
| 중점지원분야 | 가산 |
| 기술 보유 | 가산 |

즉, 화면에 표시되는 AI 승인 가능성은 `RandomForest 예측값 + 정책 룰 기반 보정값`입니다.

## 5. STEP 3: TOP3 비교 및 추천점수 부여

TOP3 추천은 `app/app.py`의 `POLICY_FUNDS`에 정의된 정책자금 목록을 순회하며 계산합니다. 현재 주요 대상 자금은 창업기반지원자금, 청년전용창업자금, 개발기술사업화자금, 신시장진출지원자금, 혁신성장지원자금, 제조현장스마트화자금, 긴급경영안정자금, 재도약지원자금 등입니다.

### 5-1. 추천점수 구성

각 정책자금의 최종 추천점수는 세 요소를 조합합니다.

| 요소 | 함수 | 설명 |
| --- | --- | --- |
| 과거 지원 패턴 점수 | `get_historical_score()` | SQLite DB에 적재된 공공데이터에서 업종·매출·업력별 지원금액 분포를 읽어 계산 |
| 목적/정책 적합 점수 | `purpose_match_score()` | 희망 자금 종류, 기술, 수출, 스마트공장, 재도약 등 매칭 가산 |
| 자격 조건 판정 | `check_required()` | 필수 조건 충족, 제한 사유, 주의사항을 판정 |

### 5-2. 과거 지원 패턴 점수

과거 지원 패턴 점수는 다음 비중으로 계산됩니다.

```text
과거 지원 패턴 점수 = 업종 점수 40% + 매출 규모 점수 30% + 업력 점수 30%
```

각 점수는 정책자금 키워드와 공공데이터의 `구분` 컬럼을 매칭한 뒤, 해당 지원금액이 데이터 내 최소~최대 범위에서 어느 정도 위치하는지 0~100으로 정규화합니다.

### 5-3. 목적/정책 적합 점수

다음 조건들이 목적/정책 적합 점수에 가산됩니다.

| 조건 | 효과 |
| --- | --- |
| 희망 자금 종류가 정책자금 선호 목적과 일치 | 가산 |
| R&D 투자자금과 기술개발 목적 일치 | 가산 |
| 기술 보유 + 기술 관련 자금 | 가산 |
| 수출기업 + 신시장진출 자금 | 가산 |
| 제조업 + 청년전용창업자금 | 가산 |
| 중점지원분야 | 가산 |
| 비수도권 | 가산 |
| 신규 고용 10인 이상 | 가산 |
| 매출 성장률 20% 이상 | 가산 |
| 수출실적 50만 달러 이상 + 신시장 자금 | 가산 |
| IP 보유 + 기술/혁신 자금 | 가산 |
| 스마트공장 도입 + 스마트화 자금 | 가산 |
| 사업전환/재창업 + 재도약 자금 | 가산 |
| 희망금액 5억 이하 | 소액 신청 가산 |

### 5-4. 추천상태 결정

각 자금은 제한 사유와 적합 점수에 따라 세 가지 상태 중 하나로 분류됩니다.

| 상태 | 조건 | 점수 산식 |
| --- | --- | --- |
| 우선 추천 | 제한 사유 없음 | `과거점수 * 0.7 + 목적점수 + 충족조건수 * 3`, 최대 100 |
| 조건부 검토 | 제한 사유 1~2개이고 목적점수 있음 | `과거점수 * 0.45 + 목적점수`, 최대 70 |
| 조건 불충족 | 제한 사유가 많거나 적합성이 낮음 | `과거점수 * 0.25`, 최대 40 |

이후 `우선 추천 -> 조건부 검토 -> 조건 불충족` 순으로 묶고, 각 그룹 안에서 추천점수 내림차순 정렬 후 상위 3개만 화면에 표시합니다.

TOP3 비교 화면에서는 다음 정보를 보여줍니다.

| 표시 항목 | 설명 |

| 추천 상태 | 우선 추천, 조건부 검토, 조건 불충족 |

| 공식 한도 | 정책자금별 대출 한도 |

## 6. STEP 4: 공고 신청 연결

공고 신청 단계는 실제 신청 API를 호출해 접수까지 처리하는 구조는 아닙니다. 현재는 추천된 정책자금별 검색 키워드를 이용해 중소벤처24 정책사업 검색 페이지로 사용자를 이동시키는 방식입니다.

### 6-1. 이동 방식

`make_search_url(keyword)` 함수가 정책자금의 `검색키워드`를 URL 인코딩하여 중소벤처24 검색 URL을 생성합니다.

URL 구조는 다음과 같습니다.

```text
https://www.smes.go.kr/main/sportsBsnsPolicy?srchGubun=3&srchText={검색키워드}&progress=ok&newView=new&cntPerPage=20
```

화면에서는 Streamlit의 `st.link_button()`을 사용해 다음과 같은 버튼을 제공합니다.

```text
{검색키워드} 공고 확인하기 (중소벤처24)
```

사용자가 버튼을 누르면 해당 정책자금 키워드로 검색된 중소벤처24 공고 목록 화면으로 이동합니다.

### 6-2. 현재 한계

현재 코드는 실제 최신 공고 API를 호출해 신청기간, 접수상태, 상세 URL을 자동 조회하는 로직을 아직 구현하지 않았습니다. 향후 공고 API 연동 시 중소벤처24 인증은 `SMES24_OPENAPI_TOKEN`을 사용합니다. 따라서 현재 공고 신청 단계는 다음 수준입니다.

| 현재 구현 | 향후 확장 가능 방향 |
| --- | --- |
| 정책자금 키워드 기반 중소벤처24 검색 링크 이동 | API 응답 기반 최신 공고 자동 조회 |
| 사용자가 검색 결과에서 직접 공고 확인 | 추천 자금별 접수상태, 신청기간, 상세 URL 직접 표시 |
| 신청 사이트로 외부 이동 | 신청 준비 체크리스트, 서류 안내, API 연계 |

## 7. 현재 추천 로직의 성격

현재 FundPilot_SYM의 추천은 완전한 실제 심사 모델이 아니라, 데모 목적의 하이브리드 방식입니다.

| 구성 | 성격 |
| --- | --- |
| 공공데이터 기반 점수 | 실제 지원 실적 분포를 활용한 패턴 점수 |
| 정책자금 조건 룰 | 주요 융자 제한/우대 조건을 코드로 반영 |
| RandomForest 모델 | 공공데이터 분포와 정책 심사 규칙을 반영한 합성 데이터 기반 예측 |
| 승인 가능성 보정 | 하드 블록과 우대 조건을 반영한 후처리 |
| 공고 연결 | 실제 공고 검색 페이지로 이동하는 링크 방식 |

따라서 화면의 분석 결과는 실제 신청 자격 확정이 아니라, 기업이 어떤 정책자금에 가까운지 빠르게 탐색하기 위한 참고용 추천 결과입니다. 실제 신청 전에는 중진공 공식 공고와 세부 제한기업 기준을 반드시 확인해야 합니다.
