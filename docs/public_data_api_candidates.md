# FundFilot_Demo 추가 공공데이터 및 Open API 후보

이 문서는 FundFilot_Demo를 실제 정책금융 추천 플랫폼으로 확장하기 위해 추가 확보하면 좋은 공공데이터포털 데이터/API 후보를 정리한 문서입니다.

중소벤처기업부 제공 데이터는 더 이상 안정적으로 관리되지 않는 것으로 판단하여 이 문서의 후보 목록에서 제외합니다. 현재 문서는 중소벤처기업진흥공단 및 금융위원회 데이터 중심으로만 정리합니다.

## 선정 기준

| 기준 | 설명 |
| --- | --- |
| 현재 입력 항목과 연결 가능 | 기업명, 사업자등록번호, 재무정보, 자산, 종업원 수, 수출, 스마트공장 등 현재 입력값과 직접 연결되는 데이터 |
| 추천점수 개선 가능 | 업종, 매출, 업력 외에 자산, 종업원 수, 시설/운전자금, 세부 자금별 수혜 패턴을 보강할 수 있는 데이터 |
| 하드코딩 제거 가능 | 융자제외 업종, 자금종류별 실적, 지역별 실적처럼 코드 내 고정값을 DB 기반으로 대체할 수 있는 데이터 |
| 리스크 분석 확장 가능 | 지역, 자금종류, 세부자금, 재창업, 수출, 스마트공장 등 수혜 패턴 이탈 분석에 활용 가능한 데이터 |

## 7. 일반 Open API로 확보하기 어려운 데이터

아래 항목은 실제 정책자금 심사에서 중요하지만, 일반 공공데이터 Open API로 바로 조회하기 어렵거나 개인정보/신용정보 동의가 필요한 영역입니다.

| 데이터 | 현재 권장 방식 |
| --- | --- |
| 국세·지방세 체납 여부 | 사용자 입력 또는 증빙 업로드 기반 |
| 신용정보원 불량정보 | 사용자 입력 또는 금융/신용정보 동의 기반 API 확보 후 연동 |
| 실제 중진공 개별 수혜 이력 | 사용자 입력, SIMS 확인값 입력, 향후 동의 기반 연동 |
| 정책자금 신청 탈락/포기 이력 | 사용자 입력 또는 내부 서비스 이용 이력 축적 |
| 기업별 실시간 신용등급 | 사용자 입력 또는 별도 신용평가사/금융 API 계약 필요 |

## 8. 추천 연동 순서

실제 개발 순서는 다음을 추천합니다.

1. `정책자금 융자제외 대상 업종` DB 기반 판정
   - 현재 `EXCLUDED_KSIC_PREFIXES` 하드코딩 제거
2. `정책자금 종업원규모별 지원 현황`, `자산 규모별 지원현황` 적재
   - 변화한 기업 입력 항목을 신청 적합도 모델 feature로 확장
3. `금융위원회_기업기본정보`, `금융위원회_기업 재무정보` 연동
   - 기업명, 사업자등록번호, 법인등록번호 기반 기업정보 자동 보강 및 입력 검증
4. 특화자금 데이터 연동
   - 청년창업, 스마트공장, 소부장, Net-Zero, 수출, 재창업 자금 추천 정교화
5. 현재 보유 데이터 자동갱신 작업 추가
   - `data/` 원천파일 수동 교체를 줄이고 DB 적재 파이프라인 안정화
