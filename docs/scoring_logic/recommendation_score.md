# 추천점수 계산 방식

이 문서는 FundPilot_SYM의 정책자금 추천점수가 어떻게 계산되는지 정리합니다.

추천점수는 실제 심사 확률이 아닙니다. 기업 입력값과 정책자금별 조건, 공공데이터 지원 패턴, 저장된 API 패턴을 조합해 “해당 기업에 얼마나 잘 맞는 정책자금인지”를 0~100점으로 표현하는 적합도 점수입니다.

## 1. 호출 위치

추천점수는 `app/app.py`의 `recommend_fund()`에서 계산합니다.

```text
app/app.py
└─ recommend_fund(...)
   ├─ check_required(...)
   ├─ get_historical_score(...)
   ├─ purpose_match_score(...)
   ├─ api_pattern_context(...)
   └─ policy_fund_reference(...)
```

추천 후보 정책자금은 `docs/policy_fund/policy_fund_detail.md`의 `요약 내용` 표를 읽어 생성합니다. 후보별 필수조건과 선호 목적은 `app/app.py`의 `FUND_RULES`에서 보강합니다.

## 2. 최종 산식 개요

각 정책자금마다 기본점수를 계산한 뒤 API 보정을 더합니다.

```text
최종 추천점수 = 기본점수 + API보정
```

최종점수는 `0 ~ 100` 범위로 제한하고, 소수점 1자리로 반올림합니다.

```text
추천점수 = round(max(0, min(기본점수 + API보정, 100)), 1)
```

`recommend_fund()`는 이 최종 산식을 바꾸지 않고, 사용자 화면 설명을 위해 다음 구성요소를 결과 DataFrame에 함께 남깁니다.

| 컬럼 | 의미 |
| --- | --- |
| `과거지원패턴점수` | 업종·매출·업력 공공데이터 패턴 기반 점수 |
| `목적정책적합점수` | 희망 목적, 기술, 수출, 중점분야 등 정책 목적 매칭 점수 |
| `자격우대충족건수` | `우선 추천` 기본점수에 반영되는 충족·우대 조건 건수 |
| `기본추천점수` | 추천상태별 산식으로 API 보정 전 계산한 점수 |
| `API보정` | 저장된 KOSMES/API 패턴이 최종점수에 가감한 점수 |
| `API근거` | API 보정을 설명하는 긍정 근거 |
| `패턴리스크` | API 패턴상 낮게 나타난 보조 리스크 |
| `추천점수근거` | 간단한 요약 설명. 현재 화면에는 직접 표시하지 않음 |
| `추천점수상세근거` | STEP 3 expander에 표시하는 상세 설명 |

## 3. 추천상태별 기본점수

기본점수 산식은 자격조건 판정 결과에 따라 달라집니다.

| 추천상태 | 조건 | 기본점수 산식 |
| --- | --- | --- |
| 우선 추천 | 불충족 사유 없음 | `과거지원패턴점수 * 0.7 + 목적적합점수 + 충족조건수 * 3`, 최대 100 |
| 조건부 검토 | 불충족 사유 1~2개이고 목적적합점수 있음 | `과거지원패턴점수 * 0.45 + 목적적합점수`, 최대 70 |
| 조건 불충족 | 그 외 | `과거지원패턴점수 * 0.25`, 최대 40 |

그 뒤 API보정을 더해 최종 추천점수를 만듭니다.

## 4. 과거지원패턴점수

`get_historical_score()`가 계산합니다.

```text
과거지원패턴점수 =
업종 점수 * 0.4
+ 매출 규모 점수 * 0.3
+ 업력 점수 * 0.3
```

각 점수는 SQLite DB에 적재된 기존 CSV 기반 공공데이터에서 구합니다.

| 점수 | 참조 데이터 |
| --- | --- |
| 업종 점수 | `정책자금 업종별 지원` |
| 매출 규모 점수 | `매출액 규모별 지원실적` |
| 업력 점수 | `정책자금 업력별 지원` |

정책자금의 `broad_keywords`, `category`, `name`을 데이터의 `구분` 컬럼과 매칭하고, 해당 지원금액이 전체 데이터의 최소~최대 범위에서 어느 위치인지 0~100점으로 정규화합니다.

매칭할 수 없거나 데이터가 없으면 기본값 50점을 사용합니다.

## 5. 자격조건 판정

`check_required()`가 정책자금별 필수조건, 제한 사유, 주의사항, 충족조건을 판정합니다.

### 5-1. 공통 제한 사유

다음 조건은 정책자금 전반의 제한 또는 감점 요인으로 작동합니다.

| 조건 | 처리 |
| --- | --- |
| 국세·지방세 체납 | 불충족 사유 |
| 휴업 또는 폐업 | 불충족 사유 |
| 신용정보 제한 | 불충족 사유 |
| 융자제외 업종 | 불충족 사유 |
| 코스피 상장 또는 코스닥 일반상장 제한 | 불충족 사유 |
| 신용등급 BBB 이상 우량기업 제한 | 불충족 사유 |
| 자본총계 200억 초과 또는 자산총계 700억 초과 | 불충족 사유 |
| 업력 5년 초과 한계기업 위험 | 불충족 사유 |
| 부채비율 기준 초과 | 예외가 없으면 불충족 사유 |
| 정부·지자체 정책자금 200억 초과 | 불충족 사유 |
| 기존 대출잔액 + 희망금액 60억 초과 | 불충족 사유 |

주의사항은 추천상태를 직접 실패로 만들지는 않지만 화면 경고와 리스크 설명에 사용됩니다.

### 5-2. 자금별 필수조건

`FUND_RULES`의 `required` 값에 따라 추가 판정합니다.

| required 키 | 의미 |
| --- | --- |
| `max_years` | 최대 업력 조건 |
| `min_years` | 최소 업력 조건 |
| `max_ceo_age` | 대표자 최대 연령 조건 |
| `tech_required` | 기술사업화 요건 |
| `smart_factory_required` | 스마트공장 요건 |
| `carbon_required` | Net-Zero·탄소중립 요건 |
| `business_conversion_required` | 사업전환 요건 |
| `restart_required` | 재창업 요건 |
| `trade_damage_required` | 통상변화 피해 요건 |
| `management_distress_required` | 경영애로 요건 |
| `disaster_required` | 재해 피해 요건 |
| `export_stage` | 수출 단계 조건 |

충족된 조건은 `matched` 목록에 들어가며, `우선 추천` 상태에서는 `충족조건수 * 3`으로 기본점수에 가산됩니다.

## 6. 목적적합점수

`purpose_match_score()`가 기업의 희망 자금 목적과 정책자금 특성을 비교해 가산점을 계산합니다.

| 조건 | 가산점 |
| --- | --- |
| 희망 자금 종류와 정책자금 선호 목적이 일치 | +20 |
| `R&D 투자자금` 선택 + 정책자금 선호 목적에 `기술개발` 포함 | +12 |
| 기술 보유 + 정책자금명에 `기술` 포함 | +10 |
| 수출 단계가 내수기업이 아니고 신시장/수출/글로벌 자금 | +10 |
| 제조업 + 청년전용창업자금 | +5 |
| 혁신성장·중점지원 분야 | +8 |
| 비수도권 | +4 |
| 최근 1년 신규 고용 10인 이상 | +6 |
| 3개년 매출 성장률 20% 이상 | +5 |
| 수출실적 50만 달러 이상 + 신시장/수출/글로벌 자금 | +6 |
| 지식재산권 1건 이상 + 기술/혁신 자금 | +5 |
| 스마트공장 도입 + 스마트화 자금 | +12 |
| Net-Zero·탄소중립 관련 + 관련 자금 | +12 |
| 재해 피해 + 재해중소기업지원 | +14 |
| 경영애로 + 일시적경영애로 또는 구조개선 | +10 |
| 통상변화 피해 + 통상변화대응 | +12 |
| 재창업 + 재창업자금 | +12 |
| 사업전환·재창업 상태 + 재도약 계열 | +12 |
| 희망금액 5억 이하 | +2 |

## 7. API보정

`api_pattern_context()`가 저장된 API/DB 패턴을 이용해 보정합니다.

### 7-1. 정량 패턴 보정

다음 패턴 점수를 읽어 50점 대비 높거나 낮은 정도를 보정합니다.

| 패턴 | 가중치 | 참조 데이터 |
| --- | --- | --- |
| 종업원규모 | `(점수 - 50) * 0.05` | 중진공 종업원규모별 지원 현황 |
| 자산규모 | `(점수 - 50) * 0.05` | 중진공 자산규모별 지원 현황 |
| 자금종류 전환율 | `(점수 - 50) * 0.07` | 자금종류별 신청·지원결정·대출 전환율 |
| 지역 실행률 | `(점수 - 50) * 0.04` | 지역 신청·추천·대여 실행률 |

패턴 점수가 65점 이상이면 API 근거로, 35점 이하이면 패턴 리스크로 기록합니다.

### 7-2. 특화 API 보너스

저장된 특화 API 테이블이 있고 기업 조건이 맞으면 추가 보너스를 더합니다.

| 조건 | 보너스 |
| --- | --- |
| 대표자 만 39세 이하 + 청년전용창업자금 | +4 |
| 스마트공장 보유 + 스마트화 자금 | +5 |
| Net-Zero 조건 + Net-Zero/혁신/신성장 계열 | +3 |
| 중점지원분야 + 혁신성장지원 | +4 |
| 수출 단계 보유 + 신시장/수출/글로벌 계열 | +4 |
| 소부장 조건 + 혁신/신성장 계열 | +3 |
| 사업전환 조건 + 재도약/사업전환/사업재편 계열 | +3 |

API보정 최종값은 `-10점 ~ +14점` 범위로 제한됩니다.

## 8. 정렬과 TOP3

모든 정책자금 후보에 대해 추천점수를 계산한 뒤, `추천점수` 내림차순으로 정렬합니다.

```text
result.sort_values("추천점수", ascending=False, kind="mergesort")
```

상위 3개만 화면에 표시합니다. 같은 점수일 때는 stable sort인 `mergesort`를 사용하므로 기존 후보 순서가 유지됩니다.

## 9. 화면 표시

STEP 2 분석결과 화면은 1위 정책자금의 추천점수를 대형 숫자와 게이지로 보여줍니다.

STEP 2 추천 분석 결과 영역은 최우선 추천 자금의 추천점수, 추천상태, 주요 조건만 보여주며 추천점수 산정 근거는 표시하지 않습니다.

STEP 3 TOP3 비교 화면은 각 정책자금의 추천점수, 추천상태, 정책자금 조건, 주의사항, 제외사유를 비교합니다. 정책자금별 `추천점수 산정 근거` expander에서 `추천점수상세근거`를 표시하고, 이어서 과거 지원 패턴, 목적/정책 적합, 자격·우대 충족 건수, 기본 추천점수, API 보정, 총합, 충족 조건을 표로 보여줍니다.

정책자금별 `상세 조건 보기` expander는 신청 조건 중심 정보만 표시합니다.
